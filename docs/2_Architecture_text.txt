§3 — Database Schema
001_init.sql — Complete
-- Migration tracking
CREATE TABLE IF NOT EXISTS migrations_applied (
  name        TEXT PRIMARY KEY,
  applied_at  TIMESTAMPTZ DEFAULT now()
);
-- Demo organisation (one row, seeded by migrate.ts)
CREATE TABLE organisations (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name        TEXT NOT NULL,
  created_at  TIMESTAMPTZ DEFAULT now()
);
-- Documents
CREATE TABLE documents (
  id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id              UUID NOT NULL REFERENCES organisations(id),
  document_type       TEXT NOT NULL
    CHECK (document_type IN ('commercial_invoice','packing_list','bill_of_lading')),
  status              TEXT NOT NULL DEFAULT 'pending'
    CHECK (status IN ('pending','processing','review','approved','failed')),
  file_path           TEXT NOT NULL,      -- relative: {doc_id}/{filename}
  original_filename   TEXT NOT NULL,
  overall_confidence  NUMERIC(5,2),       -- mean of 14 field scores
  uploaded_at         TIMESTAMPTZ DEFAULT now(),
  updated_at          TIMESTAMPTZ DEFAULT now()
);
-- Extracted fields — relational, one row per field
CREATE TABLE extracted_fields (
  id             UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  document_id    UUID NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
  org_id         UUID NOT NULL,
  field_name     TEXT NOT NULL,
  ai_value       TEXT,          -- Claude output; arrays as JSON string
  final_value    TEXT,          -- after user review
  confidence     INTEGER NOT NULL CHECK (confidence BETWEEN 0 AND 100),
  was_corrected  BOOLEAN DEFAULT false,
  created_at     TIMESTAMPTZ DEFAULT now()
);
-- Correction history — append-only audit trail
CREATE TABLE correction_history (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  document_id      UUID NOT NULL REFERENCES documents(id),
  org_id           UUID NOT NULL,
  field_name       TEXT NOT NULL,
  ai_value         TEXT,                  -- snapshot at extraction time
  corrected_value  TEXT NOT NULL,
  corrected_at     TIMESTAMPTZ DEFAULT now()
);
-- Trigger: make correction_history truly append-only
CREATE OR REPLACE FUNCTION prevent_correction_modification()
RETURNS TRIGGER AS $$
BEGIN
  RAISE EXCEPTION 'correction_history is append-only';
END;
$$ LANGUAGE plpgsql;
CREATE TRIGGER correction_immutable
  BEFORE UPDATE OR DELETE ON correction_history
  FOR EACH ROW EXECUTE FUNCTION prevent_correction_modification();

§4 — Claude Extraction Contract
System Prompt (exact text)
You are a logistics document data extraction specialist.
Extract structured data from the document provided.
Document type: {DOCUMENT_TYPE}
Return ONLY a valid JSON object using exactly the structure below.
For each field: set "value" to the extracted content, or null if not found.
Set "confidence" to an integer 0-100 reflecting your certainty.
{

  "consignee_address":     { "value": "string or null", "confidence": 0-100 },
  "commodity_description": { "value": "string or null", "confidence": 0-100 },

  "country_of_origin":     { "value": "string or null", "confidence": 0-100 },
  "declared_value":        { "value": "string or null", "confidence": 0-100 },
  "currency":              { "value": "string or null", "confidence": 0-100 },
  "incoterms":             { "value": "string or null", "confidence": 0-100 },

Output the JSON object only. No markdown, no explanation, no code fences.
Zod Schema (backend/src/types/index.ts)
import { z } from 'zod';
const FieldSchema = z.object({
  value:      z.union([z.string(), z.array(z.string()), z.null()]),
  confidence: z.number().int().min(0).max(100),
});
export const ExtractionResponseSchema = z.object({
  shipper_name:          FieldSchema,
  shipper_address:       FieldSchema,
  consignee_name:        FieldSchema,
  consignee_address:     FieldSchema,
  commodity_description: FieldSchema,
  quantity_and_units:    FieldSchema,
  gross_weight:          FieldSchema,
  net_weight:            FieldSchema,
  country_of_origin:     FieldSchema,
  declared_value:        FieldSchema,
  currency:              FieldSchema,
  incoterms:             FieldSchema,
  document_date:         FieldSchema,
  reference_numbers:     FieldSchema,
});
export type ExtractionResponse = z.infer<typeof ExtractionResponseSchema>;
// Helper: strip markdown fences before parsing
export function cleanJsonText(text: string): string {
  return text.replace(/^```json\s*/i, '').replace(/```\s*$/i, '').trim();
}

extractionService.ts — Step-by-Step
Step
Action
1
SET documents.status = "processing", updated_at = now() WHERE id = documentId
2
SELECT id, file_path, document_type FROM documents WHERE id = documentId
3
Read file buffer: fs.readFile(path.join(UPLOAD_DIR, doc.file_path))
4
Detect MIME type from buffer first 8 bytes → set media_type variable
5
Build Claude message: content array with {type:"document"|"image", source:{type:"base64", media_type, data: buf.toString("base64")}} followed by {type:"text", text: PROMPT_WITH_DOC_TYPE}
6
await anthropic.messages.create({ model: CLAUDE_MODEL, max_tokens: 2048, messages: [message] })
7
const raw = response.content[0].text
8
const cleaned = cleanJsonText(raw)   ← strips markdown fences
9
const parsed = JSON.parse(cleaned)   ← may throw SyntaxError
10
const validated = ExtractionResponseSchema.parse(parsed)   ← may throw ZodError
11
If step 9 or 10 throws → RETRY ONCE: rebuild the Claude call with an additional line at the end of the prompt: "Your previous response was invalid JSON. Return ONLY the JSON object." Repeat steps 6–10. If still fails → SET status="failed" → return.
12
Compute overall_confidence = mean of all 14 field.confidence values → round to 2dp
13
BEGIN TRANSACTION
14
UPDATE documents SET status="review", overall_confidence=X, updated_at=now()
15
INSERT INTO extracted_fields (14 rows): for each key in validated: { document_id, org_id: DEMO_ORG_ID, field_name: key, ai_value: Array.isArray(val.value) ? JSON.stringify(val.value) : val.value, final_value: same, confidence: val.confidence, was_corrected: false }
16
COMMIT

Step
Action
1
SELECT field_name, ai_value FROM extracted_fields WHERE document_id = id → build Map<field_name, ai_value>
2
BEGIN TRANSACTION

GET /api/documents — Search & Filter SQL
-- Base query (always scoped to org_id)
SELECT d.id, d.document_type, d.status, d.overall_confidence,
       d.uploaded_at, d.original_filename,
       MAX(CASE WHEN ef.field_name='reference_numbers' THEN ef.final_value END) AS reference_numbers,
       MAX(CASE WHEN ef.field_name='shipper_name'      THEN ef.final_value END) AS shipper_name,
       MAX(CASE WHEN ef.field_name='consignee_name'    THEN ef.final_value END) AS consignee_name
FROM documents d
LEFT JOIN extracted_fields ef ON ef.document_id = d.id
WHERE d.org_id = $DEMO_ORG_ID
-- Applied conditionally:
AND d.document_type = $type                           -- if ?type provided
AND d.uploaded_at >= $date_from                       -- if ?date_from provided
AND d.uploaded_at <= $date_to                         -- if ?date_to provided
-- Country filter (subquery on extracted_fields):
AND d.id IN (
  SELECT document_id FROM extracted_fields
  WHERE field_name = 'country_of_origin' AND final_value = $country
)
-- Search filter (subquery on extracted_fields):
AND d.id IN (
  SELECT document_id FROM extracted_fields
  WHERE field_name IN ('shipper_name','consignee_name','commodity_description','reference_numbers')
  AND final_value ILIKE $q_pattern    -- $q_pattern = '%' || q || '%'
)
GROUP BY d.id
ORDER BY d.uploaded_at DESC
LIMIT $limit OFFSET $offset

§9 — Critical Implementation Rules
#
Rule
1
NO AUTH ANYWHERE. No Authorization header checks, no JWT, no session middleware. Every Express route is open. Every Next.js page renders without a session check.
2
org_id IS ALWAYS process.env.DEMO_ORG_ID. Never read from request body, params, or query string.
3
ZOD BEFORE EVERY DB WRITE. ExtractionResponseSchema.parse() must succeed before any INSERT into extracted_fields. Never write partial data.
4
CORRECTION_HISTORY IS APPEND-ONLY. Never UPDATE or DELETE from correction_history. The DB trigger enforces this — any attempt will throw a PostgreSQL exception.
5
FIRE AND FORGET extraction. The upload route handler calls extractionService.extractDocument(id) WITHOUT await and then returns the HTTP 201 response. Never block the HTTP response waiting for Claude.
6
POLL FOR STATUS. Frontend polls GET /api/documents/:id/status every 2 seconds until status is not "processing". Use setInterval, clear on status change or component unmount.
7
FILE PATH SANITISATION. Before writing any file: strip ../ and ./ from filename with a regex. Store only the relative path {documentId}/{sanitised_filename} in the DB.
8
reference_numbers STORAGE. Always JSON.stringify(array) before storing in ai_value or final_value. Always JSON.parse(string) when reading back. Never store a raw JS array.
9
WRAP CLAUDE IN TRY/CATCH. Every code path through extractionService is wrapped. On any error: set status="failed", log the error with console.error, return. Never re-throw to the Express unhandled rejection handler.
10
TRANSACTION INTEGRITY. extractionService and correctionService both use BEGIN/COMMIT. If any statement in the transaction throws, ROLLBACK and re-throw to the calling catch block — which sets status="failed".
